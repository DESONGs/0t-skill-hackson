# 0T-Skill 项目介绍

## 项目背景

链上交易世界中存在大量风格鲜明的钱包地址——有的是高频剥头皮交易者，有的是长线价值持有者，有的是 Memecoin 猎手。这些交易风格蕴含着有价值的策略模式，但传统方式难以系统化地提取和复用。

**0T-Skill** 解决的核心问题是：**如何将任意链上地址的交易行为自动蒸馏为结构化、可执行的交易策略 Skill，并通过标准化执行层完成链上交易闭环。**

## 核心流程

### 输入

用户提供一个链上钱包地址和目标链（如 BSC、Ethereum），系统自动完成后续全流程。

### 六模块蒸馏管线

```text
┌─────────────┐
│ 输入：钱包地址 │
└──────┬──────┘
       │
┌──────▼──────┐
│ M1 数据采集   │  ← AVE API（并行化）
│ 与标准化      │     inspect_wallet / inspect_token / inspect_market / review_signals
└──┬───┬───┬──┘
   │   │   │
   ▼   ▼   ▼         三者并行
┌──────┐ ┌──────┐ ┌──────┐
│ M2   │ │ M3   │ │ M4   │
│交易配对│ │市场   │ │信号与 │
│与统计  │ │上下文 │ │风控   │
└──┬───┘ └──┬───┘ └──┬───┘
   │        │        │
   └────────┼────────┘
            │
   ┌────────▼────────┐
   │ M5 LLM 蒸馏     │  ← Pi/Kimi 反射式推理
   │ 与 Skill 生成    │
   └────────┬────────┘
            │
   ┌────────▼────────┐
   │ M6 回测与        │
   │ 置信度评估        │
   └────────┬────────┘
            │
   ┌────────▼────────────┐
   │ 执行适配层            │  ← OKX OnchainOS CLI
   │ dry-run / live       │
   └────────┬────────────┘
            │
   ┌────────▼────────┐
   │ 输出：Skill 包   │
   │ + 策略画像       │
   └─────────────────┘
```

### 各模块职责

| 模块 | 职责 | 数据来源 |
|------|------|---------|
| M1 数据采集与标准化 | 并行拉取钱包画像、代币详情、市场数据、信号 | AVE API |
| M2 交易配对与统计特征 | FIFO 买卖配对、胜率/盈亏比/持仓周期计算 | M1 产出的历史交易流水 |
| M3 市场环境上下文 | BTC/ETH 宏观状态、焦点代币动量/波动率 | AVE 市场数据 |
| M4 触发信号与风控过滤 | 入场因子频率统计、合约风险过滤 | M1 产出 + M2 统计 |
| M5 LLM 蒸馏与 Skill 生成 | 结构化反射生成交易风格画像、策略规格、Skill 包 | M1-M4 汇总的 compact_input |
| M6 回测与置信度评估 | 信号回放验证、多维置信度评分 | M2 配对交易 + M3 市场上下文 |

### 输出

一个标准化的 **Skill 包**，包含：

- **交易风格画像**（`style_profile`）：执行节奏、风险偏好、确信度画像、活跃时段
- **策略规格**（`strategy_spec`）：入场/出场条件、仓位管理、风控规则
- **执行意图**（`execution_intent`）：执行适配器、工作流、预检清单
- **可执行脚本**：`primary.py`（策略决策）+ `execute.py`（交易执行）

## 示例 Skill 详解

本仓库包含两个真实地址蒸馏生成的 Skill 包，展示了系统对不同交易风格的蒸馏能力。

### 示例 1：BSC Memecoin Scalper

**源地址**：`0x9048f6c683abb0eba156797fd699fe662b4dbfef`（BSC 链）

#### 蒸馏结果

| 维度 | 蒸馏输出 |
|------|---------|
| 风格标签 | BSC Memecoin Scalper |
| 执行节奏 | 超短剥头皮（同分钟爆发，平均持仓 21 秒） |
| 风险偏好 | 激进 Memecoin 交易者 |
| 确信度画像 | 动量驱动的盘口阅读者，钻石手回撤容忍 |
| 稳定币偏好 | 零稳定币配置，全仓投入 |
| 活跃时段 | europe-overlap（12:00-18:00 UTC） |

#### 统计特征

| 指标 | 数值 |
|------|------|
| 总交易数 | 149 笔 |
| 已配对交易 | 95 笔 |
| 胜率 | 65.3% |
| 盈亏比 | 2.34 |
| 每笔预期收益 | $22.45 |
| 平均持仓时间 | 21 秒 |
| 最大回撤 | -61.3% |
| 加仓模式 | 金字塔（平均 1.5 次分批） |
| 持仓分类 | scalping（剥头皮） |

#### 策略规则（蒸馏产出）

**入场条件**：
1. BSC Memecoin 焦点代币出现成交量飙升（权重 40%）
2. Europe-overlap 时段对齐（权重 30%）
3. 代币价格 < $0.01 且有 WBNB 报价流动性（权重 30%）

**出场条件**：
- 时间止损：21 秒平均持仓，同分钟爆发意味着即时退出
- 回撤容忍：钻石手标签允许 -61% 回撤
- 目标倍数：剥头皮获取小额绝对收益（~$22 每笔预期）

**风控规则**：
- 硬性拦截 `owner_can_change_transfer_mode` 标记的代币
- 单笔暴露上限 ~$580（约余额的 12%）
- 入场前验证 WBNB 流动性深度 > $1K

#### 执行意图

```json
{
  "adapter": "onchainos_cli",
  "mode": "dry_run_ready",
  "preferred_workflow": "swap_execute",
  "preflight_checks": ["security_token_scan"],
  "route_preferences": ["WBNB"],
  "split_legs": true,
  "requires_explicit_approval": true
}
```

### 示例 2：BSC Microcap Day-Scalper

**源地址**：`0xbac453b9b7f53b35ac906b641925b2f5f2567a89`（BSC 链）

#### 蒸馏结果

| 维度 | 蒸馏输出 |
|------|---------|
| 风格标签 | BSC Microcap Day-Scalper |
| 执行节奏 | 同分钟爆发日内交易（~3.8 小时平均持仓） |
| 风险偏好 | 极端微型市值交易者 |
| 确信度画像 | 钻石手金字塔累积者 |
| 稳定币偏好 | 零稳定币，全额部署 |
| 活跃时段 | us-session（18:00-24:00 UTC） |
| 偏好代币 | XMONEY, PP, PIZZA |
| 仓位管理 | 初始试探 <3%（~$607 中位数），最大 ~$3.5K，平均 2.95 次金字塔分批 |
| 置信度 | 0.9（high） |

#### 两个示例的对比

| 对比维度 | Scalper（示例 1） | Day-Scalper（示例 2） |
|---------|-----------------|---------------------|
| 持仓时间 | 21 秒 | 3.8 小时 |
| 回撤容忍 | -61% | -85% |
| 金字塔分批 | 1.5 次 | 2.95 次 |
| 置信度 | 0.5（medium） | 0.9（high） |
| 活跃时段 | 欧洲盘 | 美盘 |
| 仓位比例 | 4.3%-11.9% | <3% 初始试探 |

这两个示例展示了系统能够区分不同交易风格的细微差异——即使两者都活跃在 BSC 链的微型市值代币上，系统仍然蒸馏出了截然不同的执行节奏、仓位管理和风控参数。

## 项目定位

0T-Skill 不是一个通用量化交易框架，而是一个 **交易风格蒸馏引擎**。它的价值在于：

1. **从行为到策略**：将链上地址的被动交易记录，转化为主动可执行的结构化策略
2. **数据驱动的蒸馏**：统计特征（胜率、盈亏比、持仓周期）而非主观判断驱动策略生成
3. **LLM 增强的分析**：利用大模型的推理能力对统计特征进行结构化审查与策略抽象
4. **标准化 Skill 输出**：产出符合 Skill-OS 标准的可执行包，可直接接入 OKX OnchainOS 执行层
5. **闭环验证**：内置回测与置信度评估，量化策略质量后再决定执行模式（dry-run / live）
