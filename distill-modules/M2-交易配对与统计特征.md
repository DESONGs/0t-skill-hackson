# M2 - 交易配对与统计特征模块

## 模块定位

对应 harness 中的「统计学特征输入 (Statistical Metadata)」和「蒸馏架构 - 数据清洗与配对」。本模块将原始交易流水配对为完整交易闭环（Buy→Sell），并计算胜率、盈亏比、持仓周期等统计指标。

**这是当前代码库最大的缺口之一。**

### 完全缺失 ✗

| harness 需求 | 描述 | 影响 |
|-------------|------|------|
| **Buy-Sell 配对** | 将同一 Token 的 buy 和 sell 匹配为完整交易闭环 | 没有配对就无法算胜率/盈亏比 |
| **持仓周期 (Holding Period)** | 从买入到卖出的时间差 | 无法判断 Scalping / Day / Swing |
| **胜率与盈亏比** | 每笔完整交易的 P&L 统计 | 蒸馏核心指标缺失 |
| **回撤容忍度** | 浮亏达到多少时止损 | 无法生成止损规则 |
| **加仓逻辑检测** | 同一 Token 多笔买入的价格/金额模式 | 无法识别马丁格尔等策略 |
| **盈亏归因** | 过滤空投等被动收益，只分析主动交易 | 统计被噪声污染 |

## 核心算法设计

### 1. 交易配对引擎 (Trade Pairing)

**FIFO 配对法**：同一 Token 的买入按时间排序，用先进先出匹配卖出。

```python
@dataclass
class CompletedTrade:
    token_symbol: str
    token_address: str
    buy_timestamp: datetime
    sell_timestamp: datetime
    buy_amount_usd: float
    sell_amount_usd: float
    holding_seconds: int
    pnl_usd: float
    pnl_pct: float
    is_profitable: bool

@dataclass
class OpenPosition:
    token_symbol: str
    token_address: str
    buy_timestamp: datetime
    buy_amount_usd: float
    classification: str  # "long_hold" | "unrealized" | "possible_rug"

def pair_trades(activities: list[dict]) -> tuple[list[CompletedTrade], list[OpenPosition]]:
    """
    FIFO 配对算法。
    - 按 token_address 分组
    - 同一 token 内按时间排序
    - buy 入队列，sell 从队头匹配
    - 未匹配的 buy 归为 OpenPosition
    """
    buy_queues: dict[str, deque] = defaultdict(deque)
    completed = []
    
    sorted_activities = sorted(activities, key=lambda x: x["timestamp"])
    for activity in sorted_activities:
        token_addr = activity.get("token_address", "").lower()
        if not token_addr:
            continue
        action = activity.get("action", "").lower()
        
        if action == "buy":
            buy_queues[token_addr].append(activity)
        elif action == "sell" and buy_queues[token_addr]:
            buy = buy_queues[token_addr].popleft()
            # ... 计算 PnL, holding_seconds 等
            completed.append(CompletedTrade(...))
    
    open_positions = [
        OpenPosition(...)
        for addr, queue in buy_queues.items()
        for buy in queue
    ]
    return completed, open_positions
```

### 3. 加仓模式检测

```python
def detect_averaging_pattern(buy_entries: list[dict]) -> str:
    """
    同一 token 多笔买入：
    - 金额递增 → martingale（马丁格尔）
    - 金额恒定 → linear_dca（定额定投）
    - 金额递减 → pyramid（金字塔加仓）
    - 单笔买入 → none
    """
    if len(buy_entries) <= 1:
        return "none"
    amounts = [e["amount_usd"] for e in buy_entries]
    diffs = [amounts[i+1] - amounts[i] for i in range(len(amounts)-1)]
    if all(d > 0 for d in diffs):
        return "martingale"
    if all(abs(d) < amounts[0] * 0.15 for d in diffs):
        return "linear_dca"
    if all(d < 0 for d in diffs):
        return "pyramid"
    return "mixed"
```

## 现有代码的问题与解决

**问题**：`WalletStyleExtractor.extract` 中的 confidence 评分只考虑数据量（activity_count），不考虑数据质量（胜率、完成交易数）。

**解决**：将 M2 的统计结果纳入 confidence 计算：

```python
# 改进后的 confidence 计算
confidence = 0.20
confidence += min(completed_trade_count, 10) * 0.05   # 完成交易数贡献
confidence += min(activity_count, 6) * 0.03            # 原有活动量（权重降低）
confidence += 0.10 if win_rate is not None else 0.0    # 有胜率数据加分
confidence += 0.08 if holding_classification != "sparse" else 0.0
confidence += 0.08 if profit_factor > 1.0 else 0.0
```

## 风险与注意事项

1. **数据不完整**：链上交易可能跨多个 DEX，AVE API 可能无法覆盖所有交易，配对结果可能有偏差
2. **复杂交易类型**：多跳 swap（A→B→C）需要特殊处理，当前简化为按主要 token 归类
3. **计算量控制**：大户可能有数千笔交易，配对算法需要 O(N) 时间复杂度，不应成为瓶颈
4. **这里的统计结果不直接传给 LLM**：只传汇总指标（几个数字），原始 completed_trades 列表仅供 M6 回测使用
