# M4 - 触发信号与风控过滤模块

## 模块定位

对应 harness 中的「触发信号逻辑 (Signal Triggers)」和「过滤因子 (Filters)」。负责从交易历史中抽象出入场规则和安全过滤器，并与 AVE 的风险 API 联动。

## 核心设计

### 1. 入场因子蒸馏器 (Entry Factor Distiller)

从 M2 的 `completed_trades`（盈利交易）中提取共性条件：

```python
@dataclass
class EntryFactor:
    factor_type: str       # "volume_spike" | "price_breakout" | "whale_inflow" | "rsi_oversold"
    description: str       # 人类可读描述
    frequency: float       # 盈利交易中出现此因子的频率 (0-1)
    avg_pnl_when_present: float  # 有此因子时的平均收益率
    confidence: float

def distill_entry_factors(
    completed_trades: list[CompletedTrade],
    market_contexts: dict[str, TokenMarketContext],
) -> list[EntryFactor]:
    """
    分析盈利交易的市场环境共性。
    
    MVP 阶段使用规则匹配而非回归分析：
    - 买入前 1h 价格 >+5% → "momentum_chase"（追涨）
    - 买入前 1h 价格 <-10% → "dip_buy"（抄底）
    - 买入时 volume/liquidity ratio >2 → "volume_spike"（放量）
    - 买入时 volatility_regime == "extreme" → "volatility_play"（波动率交易）
    """
    profitable = [t for t in completed_trades if t.is_profitable]
    if not profitable:
        return [EntryFactor(
            factor_type="insufficient_data",
            description="Not enough profitable trades to distill entry factors",
            frequency=0, avg_pnl_when_present=0, confidence=0.1
        )]
    
    factors = []
    # 检测每个因子在盈利交易中的出现频率
    for factor_type, check_fn in FACTOR_CHECKS.items():
        matching = [t for t in profitable if check_fn(t, market_contexts)]
        if matching:
            freq = len(matching) / len(profitable)
            avg_pnl = sum(t.pnl_pct for t in matching) / len(matching)
            factors.append(EntryFactor(
                factor_type=factor_type,
                description=FACTOR_DESCRIPTIONS[factor_type],
                frequency=freq,
                avg_pnl_when_present=avg_pnl,
                confidence=min(0.9, freq * 0.7 + 0.2)
            ))
    
    return sorted(factors, key=lambda f: f.frequency, reverse=True)[:5]
```

### 2. 安全过滤器 (Risk Filter)

结合 AVE 的 risk_snapshot 和 holder_snapshot 构建：

```python
@dataclass
class RiskFilter:
    filter_type: str           # "lp_lock" | "holder_concentration" | "honeypot" | "high_tax"
    description: str
    threshold: Any
    is_hard_block: bool        # True = 绝对禁止, False = 警告
    source: str                # 数据来源 API

def build_risk_filters(token_profiles: list[dict]) -> list[RiskFilter]:
    filters = []
    
    for profile in token_profiles:
        risk = profile.get("risk_snapshot", {})
        holder = profile.get("holder_snapshot", {})
        
        # Honeypot 检测
        if risk.get("honeypot"):
            filters.append(RiskFilter(
                filter_type="honeypot",
                description="Token detected as honeypot",
                threshold=True, is_hard_block=True,
                source="inspect_token.risk_snapshot"
            ))
        
        # 高税率过滤
        buy_tax = risk.get("buy_tax_bps", 0) or 0
        sell_tax = risk.get("sell_tax_bps", 0) or 0
        if buy_tax > 500 or sell_tax > 500:  # >5%
            filters.append(RiskFilter(
                filter_type="high_tax",
                description=f"Buy tax {buy_tax}bps / Sell tax {sell_tax}bps exceeds 5%",
                threshold=500, is_hard_block=True,
                source="inspect_token.risk_snapshot"
            ))
        
        # Top holder 集中度
        top_share = holder.get("top_holder_share_pct", 0) or 0
        if top_share > 50:
            filters.append(RiskFilter(
                filter_type="holder_concentration",
                description=f"Top holders control {top_share}% of supply",
                threshold=50, is_hard_block=False,
                source="inspect_token.holder_snapshot"
            ))
    
    return filters
```

### 3. 过滤器到 Skill 的映射

将过滤器转化为 Skill 中 `anti_patterns` 的具体规则：

```python
def filters_to_anti_patterns(filters: list[RiskFilter]) -> list[str]:
    """将风控过滤器转化为 skill 的 anti_pattern 规则文本"""
    patterns = []
    for f in filters:
        if f.is_hard_block:
            patterns.append(f"BLOCK: {f.description}")
        else:
            patterns.append(f"WARN: {f.description}")
    return patterns
```

## 与其他模块的依赖关系

- **上游**: M1（token_profiles, signals）、M2（completed_trades）、M3（market_contexts）
- **下游**: M5（entry_factors 和 anti_patterns 注入 LLM prompt 和 Skill 模板）

## 风险与注意事项

1. **入场因子可能过拟合**：样本量 <10 笔盈利交易时，提取的因子可信度低，需标注 `low_confidence`
2. **LP 锁定检查**：AVE API 当前 risk_snapshot 中未明确包含 LP lock 状态，需确认 `flags` 字段是否包含相关标签，否则此项暂时搁置
3. **回归分析 vs 规则匹配**：harness 提到「相关性分析/回归分析」，MVP 阶段简化为频率统计 + 规则匹配，后续可用 scipy/sklearn 增强
4. **过滤器不应过于严格**：初始阶段以 `is_hard_block=False`（警告）为主，避免错杀机会
