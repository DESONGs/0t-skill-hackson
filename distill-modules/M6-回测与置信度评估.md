# M6 - 回测与置信度评估模块

## 模块定位

对应 harness 中的「历史实战复盘 (Backtesting & Confidence)」。验证 M5 生成的 **StrategySpec** 在历史数据上的表现，并输出置信度评分，决定初始头寸大小。  

开发前冻结口径：
- **M6 的回测、评分、置信度全部只使用 AVE 数据**，保持与蒸馏阶段同口径。
- **onchainos CLI 不参与回测打分**，只承担后续 dry-run / live execution 验证。
- **策略是否有效** 与 **执行是否可落地** 是两个独立判断：
  - 策略有效性：M6 回测与置信度
  - 执行可达性：onchainos CLI 的 security / simulate / broadcast 流程

## 核心设计

### 1. 轻量级回测引擎

不构建完整回测框架（如 Backtrader），而是基于 M2 的 `completed_trades` 做**策略信号回放**：

```python
@dataclass
class BacktestResult:
    total_signals: int          # 策略触发了多少次
    executed_trades: int        # 实际可执行的交易数
    correct_signals: int        # 信号与实际盈利方向一致的次数
    signal_accuracy: float      # correct_signals / total_signals
    
    simulated_pnl_usd: float    # 如果按策略执行的总PnL
    actual_pnl_usd: float       # 交易员实际PnL
    pnl_capture_ratio: float    # simulated / actual（策略捕获了多少交易员的收益）
    
    max_drawdown_pct: float     # 模拟最大回撤
    sharpe_ratio: float | None  # 年化夏普比（如果有足够数据）
    
    confidence_score: float     # 综合置信度 [0, 1]
    confidence_label: str       # "high" | "medium" | "low" | "insufficient_data"


def run_backtest(
    strategy: dict,                    # M5 生成的 StrategySpec（entry/exit conditions）
    completed_trades: list[CompletedTrade],  # M2 的交易配对结果
    market_snapshots: dict,            # M3 的市场快照（有限的历史数据）
) -> BacktestResult:
    """
    对每笔历史 completed_trade：
    1. 检查策略的 entry_conditions 是否会在买入时触发
    2. 检查策略的 exit_conditions 是否与实际卖出时机一致
    3. 统计策略信号与实际交易的吻合度
    """
    signals = []
    
    for trade in completed_trades:
        # 构造该交易时间点的上下文
        trade_context = _build_trade_context(trade, market_snapshots)
        
        # 检查入场条件
        entry_triggered = _evaluate_entry(strategy["entry_conditions"], trade_context)
        
        # 检查出场条件
        exit_matched = _evaluate_exit(strategy["exit_conditions"], trade)
        
        signals.append({
            "trade": trade,
            "entry_triggered": entry_triggered,
            "exit_matched": exit_matched,
            "actual_profitable": trade.is_profitable,
        })
    
    # 计算各项指标
    correct = sum(1 for s in signals if s["entry_triggered"] and s["actual_profitable"])
    total_triggered = sum(1 for s in signals if s["entry_triggered"])
    
    return BacktestResult(
        total_signals=total_triggered,
        executed_trades=len(completed_trades),
        correct_signals=correct,
        signal_accuracy=correct / max(total_triggered, 1),
        # ... 其他指标 ...
        confidence_score=_compute_confidence(signals),
        confidence_label=_confidence_label(_compute_confidence(signals)),
    )
```

### 1.1 执行验证与回测解耦

M6 内部要明确区分两种验证：

1. **Backtest Validation**
   - 输入：`strategy + completed_trades + market_snapshots`
   - 数据源：AVE-only
   - 输出：`confidence_score / signal_accuracy / pnl_capture_ratio`
2. **Execution Readiness Validation**
   - 输入：`execution_intent + trade_plan`
   - 工具：onchainos CLI（`security`、`swap`、`gateway simulate`）
   - 输出：`dry_run_ready / live_ready / blocked_by_risk`

第二类验证不进入策略置信度打分，否则会把执行层基础设施状况和策略质量混为一谈。

### 2. 置信度综合评分

```python
def _compute_confidence(signals: list[dict]) -> float:
    """
    多维度加权置信度评分。
    
    维度：
    - 数据充分性 (30%): 样本量是否足够
    - 信号准确率 (35%): 入场条件与盈利交易的吻合度
    - PnL 捕获率 (20%): 策略能复制多少原始收益
    - 风险控制 (15%): 最大回撤是否在可控范围
    """
    n = len(signals)
    
    # 数据充分性
    data_score = min(1.0, n / 15)  # 15笔交易给满分
    
    # 信号准确率
    triggered = [s for s in signals if s["entry_triggered"]]
    correct = [s for s in triggered if s["actual_profitable"]]
    accuracy_score = len(correct) / max(len(triggered), 1)
    
    # PnL 捕获率
    sim_pnl = sum(s["trade"].pnl_usd for s in triggered if s["actual_profitable"])
    actual_pnl = sum(s["trade"].pnl_usd for s in signals if s["actual_profitable"])
    capture_score = min(1.0, sim_pnl / max(actual_pnl, 1))
    
    # 综合
    score = (
        data_score * 0.30 +
        accuracy_score * 0.35 +
        capture_score * 0.20 +
        0.15  # 风险控制基础分（需要更多数据才能精确评估）
    )
    return round(max(0.1, min(0.95, score)), 4)


def _confidence_label(score: float) -> str:
    if score >= 0.70:
        return "high"
    if score >= 0.45:
        return "medium"
    if score >= 0.25:
        return "low"
    return "insufficient_data"
```

### 3. 增强版 Smoke Test

替换现有的简单 smoke test：

```python
def enhanced_smoke_test(
    promoted_root: Path,
    preprocessed: dict,
    completed_trades: list[CompletedTrade],
    strategy: dict,
    execution_intent: dict | None = None,
) -> dict:
    """
    四阶段验证：
    1. 基础运行测试（现有 smoke test）
    2. 策略回测（新增）
    3. 执行就绪度检查（可选，onchainos CLI）
    4. 置信度评分（新增）
    """
    # Stage 1: 基础运行（保留现有逻辑）
    basic_result = _basic_smoke_test(promoted_root, preprocessed)
    if not basic_result["ok"]:
        return {**basic_result, "stage": "basic_run_failed"}
    
    # Stage 2: 回测
    backtest = run_backtest(strategy, completed_trades, {})

    # Stage 3: 执行就绪度（不计入 confidence）
    execution_readiness = None
    if execution_intent:
        execution_readiness = validate_execution_readiness(execution_intent)
    
    # Stage 4: 综合评分
    return {
        "ok": basic_result["ok"],
        "basic_test": basic_result,
        "backtest": {
            "signal_accuracy": backtest.signal_accuracy,
            "pnl_capture_ratio": backtest.pnl_capture_ratio,
            "max_drawdown_pct": backtest.max_drawdown_pct,
        },
        "confidence": {
            "score": backtest.confidence_score,
            "label": backtest.confidence_label,
        },
        "execution_readiness": execution_readiness,
        "recommended_position_size_pct": _size_from_confidence(backtest.confidence_score),
    }


def _size_from_confidence(score: float) -> float:
    """
    基于置信度推荐初始头寸大小（占总资金百分比）。
    harness 要求：AI Agent 根据置信度评分决定初始头寸大小。
    """
    if score >= 0.70:
        return 5.0   # 高置信度：5%
    if score >= 0.45:
        return 2.0   # 中等：2%
    if score >= 0.25:
        return 1.0   # 低：1%
    return 0.5        # 数据不足：0.5%
```

## 局限性与诚实评估

1. **回测 ≠ 未来表现**：历史回测无法预测未来，置信度评分仅供参考
2. **数据量限制**：大多数钱包只有几十笔交易，统计显著性不足。如果 `completed_trades < 5`，直接标记为 `insufficient_data` 而不勉强算分
3. **Market Impact 未建模**：回测假设可以按历史价格成交，不考虑滑点和市场冲击
4. **幸存者偏差**：我们只分析了交易员选择的 token，不知道他看过但没买的
5. **MVP 阶段简化**：不做 Monte Carlo 模拟、不做滚动窗口回测、不做因子归因分析。这些可以在 V2 中增加
6. **执行适配层不等于策略有效性**：即使 onchainos CLI dry-run 通过，也只能说明“可执行”，不能说明“应当执行”
7. **执行就绪依赖凭证和钱包状态**：OKX API key、钱包登录态、链上 gas 与安全扫描结果都可能阻断 live execution，这些不应污染策略评分
